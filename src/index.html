design-fetcher

Haiku Project

Connects to your Google Stitch project via MCP, pulls design mockups, and saves them locally as reference HTML/CSS assets.

Core Instructions
  • Ask for Stitch project ID.
  • Fetch design screens via Stitch MCP.
  • Save raw files to /reference/screens/.
  • Generate a summary mapping index file.
@design-fetcher fetch screens

website-builder

Sonnet Project

Translates screen mockups into production-ready semantic HTML and modular CSS code using tokens defined in your Design Strategy.

Core Instructions
  • Read design strategy and brand tokens.
  • Convert screens to responsive layouts.
  • Save clean source files to /src/.
  • Launch dev server to preview locally.
@website-builder build site

content-writer

Sonnet Project

Crafts tailored, conversational copy matching your brand voice guidelines. Eradicates corporate jargon and buzzwords.

Core Instructions
  • Extract brand voice from design strategy.
  • Formulate headlines, subheadings, and CTAs.
  • Write clear, engaging, human copy.
  • Save copy to /content/ pages.
@content-writer write copy

qa-reviewer

Haiku Project

Compares final built pages against reference mockups and strict brand compliance rules, documenting errors into a central report.

Core Instructions
  • Check code for hardcoded color tokens.
  • Validate layout matching reference screens.
  • Verify typography and grid constraints.
  • Write detailed summary to qa-report.md.
@qa-reviewer review built site

How to Orchestrate Your Pipeline

You can tell Claude Code to coordinate all agents sequentially. This chain-of-agent workflow allows design assets to pass from screen fetcher to generator to reviewer without manual intervention.

Run in order: 1. @design-fetcher fetch screens from Stitch project_id 2. @website-builder build site components 3. @qa-reviewer run visual and styling checks
