# Claude Code Agents Workspace

Welcome to your Claude Code Agents demo workspace! This project is pre-configured with four specialized agents designed to automate the process of building websites from Figma/Stitch mockups, writing copy, and reviewing the results.

We've also included a local dashboard and development server to let you preview your site and orchestrate your agents.

---

## 🚀 Quick Start in 2 Minutes

### 1. Serve the Dashboard / Preview Site
Start the local development server to open the interactive workspace dashboard:
```bash
npm run dev
```
Once running, open [http://localhost:3000](http://localhost:3000) in your browser. This dashboard visualizes your pipeline and lets you quickly copy agent commands.

### 2. Start Claude Code
Open your terminal in this directory and launch the Claude Code environment:
```bash
claude
```

### 3. Load & View Agents
In the Claude Code session, view your available agents:
```text
/agents
```
You should see: `design-fetcher`, `website-builder`, `content-writer`, and `qa-reviewer`.

---

## 🛠️ Meet Your Agents

Inside the `.claude/agents/` folder, you will find these files:

| Agent Name | Description | Recommended Prompt |
| :--- | :--- | :--- |
| **`design-fetcher`** | Connects to Google Stitch MCP, lists screens, fetches screen code, and saves them. | `@design-fetcher fetch screens` |
| **`website-builder`** | Converts fetched designs into clean semantic HTML/CSS in `/src`. | `@website-builder build site` |
| **`content-writer`** | Writes copy using your brand voice and saves it to `/content`. | `@content-writer write copy` |
| **`qa-reviewer`** | Reviews code against style guides and writes reports to `qa-report.md`. | `@qa-reviewer review built site` |

---

## 🔗 Chaining Agents (Orchestration)

You can run your agents sequentially by giving Claude Code a single orchestrated command. Copy and paste this into Claude Code:

```text
Run these agents in order:
1. @design-fetcher — fetch all screens from Stitch project [your_project_id]
2. @website-builder — convert all fetched screens into a working site
3. @qa-reviewer — review everything and write the QA report
Start the next agent only after the previous one is done.
```

## ⚙️ Refinements & Bug Fixes Made

We reviewed the initial setup and applied several key improvements for seamless execution:
* **Stitch MCP Permissions:** Removed restricted tool lists from `design-fetcher` and `website-builder` so they can automatically inherit and invoke the connected Google Stitch MCP tools.
* **QA Reviewer Capabilities:** Added the `Write`, `Glob`, and `Grep` tools to `qa-reviewer` so it can scan files and output its report to `qa-report.md` (it was previously restricted to `Read` only and couldn't save its findings).
* **Search Tools for Content Writer:** Added `Glob` and `Grep` to `content-writer` so it can locate and inspect existing copy in `/src` to ensure brand alignment.
* **Integrated Dev Server:** Added a light web server (`http-server`) running via `npm run dev` to preview the generated site.
